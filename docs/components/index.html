Dropdowns

We don't want to mess with Bootstrap, we like its foundations; so we create extra classes that play nicely with existent Bootstrap components.

Pure CSS dropdown toggle

Add the .open-on-click class to any .dropdown, .dropup or any other wrapper element with position: relative; containing the .dropdown-toggle and .dropdown-menu components to make the menu appear on click without Javascript at all. Check Bootstrap's docs about dropdowns for more info.

Dismissible dropdowns

Since these dropdowns/dropups rely on the CSS :focus state of the .dropdown-toggle, their .dropdown-menu will close as soon as you click anywhere (including .dropdown-menu items) except the .dropdown-toggle itself. But, if you want to dismiss the menu on a second click, simply add any valid element with the .dropdown-close class somewhere after the .dropdown-toggle.

MacOS' caveat

Unfortunately, MacOS uses a strange set of rules to decide whenever an element is focused or not. This affects both, Safari and Firefox on MacOS. So, for example, <button> elements will never get the :focus state.

For that matter, use an <a> element with the tabindex="0" attribute as the dropdown's .dropdown-toggle component.

Extend

If you want all of the dropdowns in a project to behave like this, without adding the extra markup (.open-on-click), simply extend them in your LESS file:


.dropdown,
.dropup {
  .open-on-click();
}

Keep it clean

Remember to remove Javascript libraries and dependencies like ui.bootstrap.dropdown or Bootstrap's jQuery plugin related to dropdowns, they are unnecessary.

Tooltips

Add the data-tooltip attribute to any element to display its value as a tooltip.

Hover on the awesome tooltip and see it.

Tooltip positions

You may also add classes to the element to customize tooltip position

Custom tooltip size

If you want a custom tooltip size, selectively apply width and add white-space: normal:


.some-element-with-tooltip::before {
  width: 120px;
  white-space: normal;
}

Tooltip styles

These data-tooltips can be customized using Bootstrap's tooltip-related LESS variables.

Strict markup

Unlike Bootstrap's Breadcrumbs, .pyb-breadcrumb requires <li> elements to have a child element; that's it: always include an element inside the list item. <a> elements with href attribute receive special styles on :hover. If you don't want an item to be actionable, use any other element tag, like <span>.

Strict markup

  • Always place <label> just after the <input>.
  • Always add the id attribute to the <input>.
  • Always add the for attribute to the <label> with same value.
  • Add the text-right class to the <label> if you need it the other side.

Floating Label

Add the .floating-label class to a <label> placed just before an <input>. or <select>. It will style both, the label and the input/select element.

The .floating-label default state is floating; this is for accessibility reasons. You will have to listen to the input/select's changes to add the .rest class to the .floating-label label.

.floating-label will make its adjacent input's HTML5 placeholder invisible.
